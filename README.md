# Алгоритмы считывания показаний стрелочных измерительных приборов

В репозитории представлены классические алгоритмы компьютерного зрения с модульной структурой для автоматического считывания показаний стрелочных приборов с круглой шкалой по видеопотоку.


**На что рассчитаны алгоритмы:**

- Считывание показаний стрелочных приборов с круглой шкалой по видеозаписи
- Поддержка трех типов приборов: спидометр, индикатор часового типа, механический безмен
- Работа в трех условиях съемки: контролируемые условия, блики, наклон камеры
- На каждом этапе обработки метод можно заменить независимо от остальных
- Не требуется GPU, работает на обычном CPU

**Когда алгоритмы могут не справиться:**

- Приборы имеют некруглую шкалу
- Прибор смещен от центра кадра или занимает малую его часть
- При размытии, переэкспонировании, частичном перекрытии шкалы или при низком освещении
- Угол съемки превышает условия, при которых проводилось исследование (5-15 градусов)


## Структура репозитория

Репозиторий разделен на три папки по типу прибора. Структура папок практически идентична, различаются только наборы видеозаписей.

```
спидометр/
индикатор_фикс/
безмен/
    ├── базовый/
    ├── базовый+clahe/
    ├── базовый+hough/
    ├── базовый+perspective/
    ├── базовый+smooth/
    ├── комбинированный/
    ├── ручная разметка/
    ├── ideal.MOV
    ├── glare.MOV
    └── *.MOV   # видеозаписи для тестирования
```

### Основные видеофайлы

Каждая папка содержит видеозаписи для тестирования. Файлы названы по условиям съемки: `ideal` – контролируемые условия, `glare` – блики, `right`, `top`, `left` – различные варианты наклона камеры. 

### Папки алгоритмов

Каждая папка содержит:
- `*.py` – реализацию алгоритма
- `ideal.csv`, `glare.csv`, `right.csv` – покадровые показания алгоритма для каждого условия съемки
- `ideal (vs human).csv`, `glare (vs human).csv`, `right (vs human).csv` – показания алгоритма каждые 15 кадров для сравнения с ручной разметкой

Все алгоритмы основаны на опорном методе; в названии папки указано, какой этап был заменен на альтернативный.

| Папка | Описание |
|---|---|
| `базовый` | Опорный метод: детекция прибора кругом Хафа, бинаризация методом Оцу, детекция стрелки контурным анализом, без постобработки |
| `базовый+clahe` | -//-, бинаризация заменена на CLAHE + Оцу |
| `базовый+hough` | -//-, детекция стрелки заменена на линии Хафа |
| `базовый+perspective` | -//-, детекция прибора заменена на метод эллипса с перспективной коррекцией |
| `базовый+smooth` | -//-, добавлена постобработка: медианный фильтр + экспоненциальное скользящее среднее |
| `комбинированный` | -//-,бинаризация заменена на комбинированную адаптивную бинаризацию |
| `ручная разметка` | Эталонные показания, размеченные вручную каждые 15 кадров для вычисления метрик |


## Требования

- Python 3.9+
- OpenCV
- NumPy

Установка зависимостей:

```bash
pip install opencv-python numpy
```

## Запуск

Все алгоритмы запускаются из командной строки. Обязательно указать путь к видеофайлу и калибровочные параметры шкалы конкретного прибора.

### Параметры

**Обязательные:**

| Параметр | Тип | Описание |
|---|---|---|
| `--min-val` | float | Минимальное значение шкалы |
| `--max-val` | float | Максимальное значение шкалы |
| `--start-angle` | float | Угол начала шкалы в градусах |
| `--end-angle` | float | Угол конца шкалы в градусах |
| `--units` | str | Единицы измерения для вывода |

**Опциональные:**

| Параметр | Описание |
|---|---|
| `--skip` | Обрабатывать каждый N-й кадр (по умолчанию 1) |
| `--radius` | Ожидаемый радиус прибора в пикселях – ускоряет детекцию |
| `-o / --output` | Путь для сохранения видео с наложенными показаниями |
| `-log` | Путь для сохранения показаний в CSV-файл |

Параметры `--start-angle` и `--end-angle` задаются в градусах координат изображения и определяются по циферблату прибора до запуска.

**Пример запуска для спидометра:**

```bash
python base_smooth.py glare.MOV --min-val 0 --max-val 180 --start-angle 228 --end-angle -48 --units km/h --skip 1
```

**Пример запуска для индикатора часового типа с сохранением видео с наложенными показаниями:**

```bash
python base_ind.py ideal.MOV --min-val 0 --max-val 99 --start-angle 90 --end-angle 89 --units mm --skip 1 -o result.mp4
```

**Пример запуска для механического безмена с сохранением лог-файла:**

```bash
python base_bes.py right.MOV --min-val 0 --max-val 22 --start-angle 90 --end-angle 85 --units kg --skip 1 -log result.csv
```
